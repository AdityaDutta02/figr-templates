The AI rewrites your sentence. Autocompletes a word you didn't choose. Replaces what you typed. You didn't ask. It just did.

"Your autocomplete replaced my word with something I didn't write. It felt like the app was putting words in my mouth."
AI shows the

The suggestion appears — greyed out, ahead of the cursor. To accept: Tab. To ignore: keep typing. The user stays in control at every step.

Gmail Smart Compose: ghost text, user decides. This is why it feels helpful — not invasive.

Takes over the entire viewport. Forces engagement before any trust has been established. Removes control before the user even knows where they are.

"I opened the page and a chat window took over everything. I just closed the tab."
Floating
bubble the
user opens.

The user opens it when they want help. On their terms. Same feature — completely different trust level. One removes control. One preserves it.

The bubble sits quietly. When the user needs it, it's there. When they don't, it's invisible.

Why It Feels Creepy 04 / 07
Crosses the line
Stays in lane
Too human.
Unexpected
action.

Your brain detects unexpected action as a threat. When software acts without being asked, it triggers the same response as someone reading over your shoulder. Creepiness isn't about the feature — it's about consent.

Google Duplex demo (2018): technically impressive, universally described as "creepy." Not because it failed — because it was too human.
Shows the
suggestion.
Waits.

The moment you add a deliberate pause — a ghost suggestion, a card to accept or dismiss — the user feels in control. Show → wait → let the user decide. Skip the wait and trust erodes immediately.

GitHub commit suggestions: shown above the input. User accepts or ignores. Helpful every time.

The Pattern That Works 05 / 07

Helpful AI
Rewrites.
Replaces.
Acts silently.

Autocomplete that replaces text. AI that edits content without being asked. Features that change things the user didn't request. Every silent action is a trust withdrawal.

Autocorrect that changes a word mid-sentence and the user doesn't notice until they read it back.
Suggests.
Highlights.
Waits for yes.

The AI suggests, the user decides. Every time. The product says: here's an option. It never says: I already did it. This is the entire difference between feeling helped and feeling controlled.

Figma AI layout suggestions: shown as an overlay the user applies or dismisses. Trust intact.

Does your AI feature make a change before the user confirms? Take over screen space without being invited? If yes — you've crossed the line.

Users don't complain about bad AI features. They just stop using the product.
